## Table 2 — Query formulation techniques

| Technique | Expected improvement vs naive search | When to use | Evidence type |
|-----------|--------------------------------------|-------------|---------------|
| **Berrypicking / iterative refinement** | Avoids saturation at surface vocabulary; each result generates new terms and sub-questions the pre-specified queries couldn't anticipate | All angles — especially when domain vocabulary is partially unknown at outset | HIGH (`berrypicking-evolving-query`, Bates 1989, 3,187 citations) |
| **Adversarial query (failures/critiques)** | Positive findings drop from 96% to 44% with pre-specification; reduces LLM confirmatory search bias | Required Query 3 every angle | HIGH (Cluster C PCR, already applied to research-quality.md) |
| **Grey literature leg** | 26.7% of relevant sources uniquely identified via grey; corrects publication bias inflation of effect sizes | Required every angle | MEDIUM/CONSENSUS (Cluster C PCR, already applied) |
| **Sensitivity over specificity** | False negatives invisible; false positives filterable at screening; 78.1% of search errors affect recall, only 21.9% precision | All queries — bias toward over-retrieval | HIGH (`recall-bias-most-common-error`) + CONSENSUS (`search-sensitivity-over-specificity`) |
| **Synonym expansion + controlled vocabulary** | ~90% of searches miss at least one term variation; dual-query expansion adds ~6.7% recall on QA benchmarks | Before writing any query: expand key terms to 3–5 synonyms/equivalents; at least one query uses a synonym variant | MEDIUM (`search-error-rate-90-percent`, `query-expansion-recall-improvement-6-7-percent`) |
| **Expert strategy switching on failure** | Experts reformulate radically on failure; novices repeat variants of the same query — iterating on failure type, not query string | When a query returns <2 new keys: diagnose failure mode (vocabulary gap? wrong index? wrong framing?) and switch strategy class | MEDIUM (`expert-strategy-switching-on-failure`) |
| **Citation chaining (backward)** | Accesses citation graph — structurally distinct from keyword search; finds foundational papers that don't rank by keyword | After finding a high-quality primary source: search for papers it cites, and papers that cite it | MEDIUM (`berrypicking-six-search-tactics`) |
| **Author search** | Finds research lineage: other work from the same group, replies, corrections, follow-ups | Narrow domains where a field is person-anchored | MEDIUM (`berrypicking-six-search-tactics`) |
| **Bug-finding reframing for verify** | Reduces overconfidence gap: default prompting produces 77% success prediction at 22% actual success rate; adversarial framing improves calibration | Verify pass — already applied (Cluster C PCR); and all synthesis quality checks | MEDIUM (`adversarial-framing-calibration-gain`, arXiv:2602.06948) |

**Conflicts with current research-wave.js:**

1. **Berrypicking vs. pre-specified queries:** Berrypicking requires queries to evolve based on what earlier queries return — pre-specifying all 5 queries before retrieval prevents this. Observed resolution: archives already show follow-up queries in practice (angle 12: queries 6-11; angle 13: queries 6-10). This informal practice should be formalized. Proposed addition to research-quality.md angle format: follow-up query allowance (see Table 3).

2. **Expert strategy-switching vs. sequential firing:** Current format fires all queries sequentially without failure-mode diagnosis. An agent that gets 0 new keys from query 4 should ask "why did this fail?" and switch vocabulary class, not just continue to query 5. This is a research-wave.js design gap. Not blocking — agents can apply this informally; adding it to procedure guidance formalizes behavior already observed in some archives.

## Step 2 — Contradiction scan

**Pair 1: Berrypicking (iterative evolution) vs. pre-specified queries**  
- `berrypicking-evolving-query` (HIGH): queries must evolve mid-search from retrieved results  
- Current research-quality.md: pre-specified 5 queries per angle format  
- Resolution: NOT a contradiction — scope resolves it. Pre-specification ensures coverage of planned categories (foundational, meta-analysis, adversarial, grey-lit). Iteration ensures depth within those categories. Both principles apply simultaneously: specify categories, iterate within them. Follow-up queries already appear in archives — practice is valid, formalization is the gap. **NOT contested.**

**Pair 2: Query expansion raises recall vs. precision dilution**  
- `query-expansion-recall-improvement-6-7-percent` (MEDIUM): dual-query expansion +6.7% recall  
- `query-expansion-precision-dilution` (MEDIUM): expansion adds wrong word-sense synonyms, reducing precision  
- Resolution: NOT a contradiction — these are consequences of the same action on different metrics. Expansion helps recall, hurts precision on ambiguous terms. The architectural solution (from `rag-recall-precision-bottleneck`, HIGH): separate recall and precision as distinct pipeline concerns with separate mechanisms. Expand first, filter later. **NOT contested — additive finding.**

**Pair 3: Adversarial review requires pre-commitment vs. adversarial query at research stage**  
- `red-team-well-developed-object-prerequisite` (consensus): adversarial review needs crystallized target  
- Adversarial query requirement in research-quality.md fires at query stage (before synthesis exists)  
- Resolution: NOT a contradiction — different processes. Adversarial SEARCH (Query 3) targets finding failure evidence — no committed position needed. Adversarial REVIEW of a synthesis (the Cluster C verify pass finding) requires a crystallized target. The research-quality.md adversarial query is search, not review. **NOT contested — scope resolves it.**

**Pair 4: MCDA frameworks provide structure but lack decision guidance**  
- `mcda-nine-frameworks-lack-guidance` (consensus): 9 of 9 frameworks don't guide the actual decision  
- `mcda-should-support-not-supplant-deliberation` (consensus): quantified rankings obscure value-laden weighting  
- Resolution: NOT a contradiction — these are complementary critiques. One says frameworks don't help you decide; the other says when they appear to decide for you, that's false. Both point toward: always surface weights explicitly, always include a recommended action. **NOT contested — additive.**

**Conclusion:** No unresolvable contradictions in Cluster B. All apparent conflicts resolve by scope.

**Pass 2 — Counter-prior stress test (applied per Step 3 counter-prior finding):**

Highest-confidence group: `search-methodology-fundamentals` — main conclusion: "iterative query evolution + recall-priority + multi-source coverage = research quality."

Counter-hypothesis: "Pre-specified, precision-oriented, single-database queries are actually more effective for AI agent research because: (a) iterative evolution introduces survivorship bias in terminology, (b) recall-priority in web-scale search floods context windows, (c) multi-source coverage is redundant when one source (arXiv) dominates the AI research field."

Assessment: (a) is not supported — berrypicking finding shows vocabulary evolution is a quality improvement, not a bias amplifier; (b) is real and is why `rag-recall-precision-bottleneck` recommends a two-stage approach (retrieve for recall, rerank for precision) rather than abandoning recall-priority; (c) arXiv dominance is real and relevant — for AI/ML research specifically, arXiv coverage is higher than traditional databases, so the "multi-source" requirement may have diminishing returns. This is a legitimate scope condition not fully addressed in the archive.

**Conclusion:** Counter-hypothesis partially holds for the AI-domain-specific context. Scope condition: recall-priority + multi-source is most important for interdisciplinary or cross-domain research; for AI/ML research where arXiv dominates, the secondary source adds less marginal value. Noted as scope condition, not as contradiction. Main findings stand within their scope.

## Step 4 — Adversarial check on user/kit assumptions

**Assumption 1: "The current 5-query format is sufficient"**  
Verdict: CHALLENGED. Berrypicking (HIGH) shows real research needs follow-up queries generated from retrieved vocabulary. The 5-query limit was a satisficing rule; archives already show 7-11 queries when follow-ups are run. Formalizing follow-up queries is warranted. Not a hard contradiction — 5 pre-specified queries is a minimum, not a cap.

**Assumption 2: "The verify pass is sufficient for quality assurance"**  
Verdict: PARTIALLY CHALLENGED. `peer-review-inter-reviewer-agreement` (HIGH, r=0.34): a single-reviewer pass is essentially random. The verify agent is structurally separate (good) and adversarially framed (now fixed per Cluster C PCR). But a single verify pass on the same model family shares training distribution with the research agent — `red-team-internal-familiarity-blindspot` (HIGH) warns this narrows the hypothesis space. Full mitigation would require a different model family for verify; current architecture is a structural compromise. No immediate fix proposed — the adversarial framing update is the practical mitigation.

**Assumption 3: "Adversarial verify catches counter-prior failures"**  
Verdict: CONTRADICTED. `hypothesis-inverted-ground-truth-15pct` (HIGH): frontier models achieve <15% on counter-prior hypotheses even when explicitly tasked with finding them. A verify agent asked "try to refute this claim" will still heavily weight its priors — it will find reasons the claim is wrong that are consistent with prior expectations, and miss reasons that require counter-prior reasoning. Structural fix: forced counter-prior pass with explicit instruction to contradict the main conclusion, added to research-synthesis.md.

**Assumption 4: "Synthesis produces an unbiased picture if sources are varied"**  
Verdict: CHALLENGED by cascade finding. `cascade-peer-review-62pct-time-reduction` (HIGH): review artifacts can be reused across stages with minimal quality loss. The risk: if this principle is misapplied in synthesis (treating Cluster C's findings as "already done, skip the check"), synthesis-level confirmation bias can enter. Safeguard: this synthesis explicitly ran the counter-prior pass and documented what Cluster C covered vs. what is new. No silent carry-over.

## Step 5 — User questions (CONTESTED and GAP)

**CONTESTED:** None. All contradictions resolved by scope.

**GAPs requiring note (not user decisions):**

1. **Iterative query formalization vs. pre-specified format:** The synthesis supports formalizing follow-up queries but there is no empirical study comparing pre-specified-only vs. iterative formats in AI agent research specifically. Berrypicking evidence is from human searchers; transfer to AI agents is unvalidated. Adding follow-up query guidance is a practical extrapolation, not an empirically tested intervention for agents.

2. **Synonym expansion cost vs. benefit:** 90% of searches miss term variations (MEDIUM); but term expansion can also introduce noise (query-expansion-precision-dilution, MEDIUM). The net effect in AI agent research context is unquantified. Adding synonym expansion as a step is supported directionally; the calibration (how many synonyms, which ones) is a design judgment.

3. **Counter-prior pass reliability:** The counter-prior pass added to research-synthesis.md forces a counter-hypothesis to be generated. But `hypothesis-inverted-ground-truth-15pct` itself shows models struggle to generate genuine counter-prior hypotheses — the forced pass may produce counter-sounding but prior-consistent outputs. This is a known limitation of the structural fix; external signal injection (actual contradicting evidence, not just a prompt instruction) would be more effective.

## Step 3.6 supplement — counter-prior pass for remaining HIGH groups (REVISIT-S-03, 2026-07-05)

**Context:** Step 3.5 Pass 2 (above) ran the counter-prior stress test on `search-methodology-fundamentals` only. Under the corrected research-synthesis.md (9k, 2026-07-05), Step 3.6 is required for every HIGH group. The remaining 4 HIGH groups are assessed here.

**Group: peer-review-quality — HIGH**
Main conclusion: Single reviewer without rubric is essentially random quality (r=0.34 inter-reviewer agreement, N=45-study meta-analysis); domain-specialist sub-reviewers are the highest-ROI intervention.

Counter-hypothesis: The r=0.34 inter-reviewer agreement reflects genuine disagreement about what constitutes quality — quality is subjectively defined, so adding more reviewers adds averaging noise rather than convergence. Improving reviewer selection is misguided when the underlying construct (quality) lacks consensus.

Assessment: PLAUSIBLE for subjective quality dimensions (novelty, significance). Does NOT hold for objective error categories — `peer-review-psychology-18pct-errors` (HIGH: 18% errors in psychology papers, 1-in-8 affect conclusions) documents specific error types (statistical, procedural, factual) where inter-reviewer agreement should be higher and often fails. These are not subjective. Scope condition added: multi-reviewer with rubric is highest-ROI for **error detection**; benefit is weaker for subjective quality judgment dimensions where genuine expert disagreement is expected.

**Verdict:** PLAUSIBLE with scope. No tier change — HIGH evidence base for objective error detection is independent of the subjectivity concern. Main conclusion stands with scope qualifier.

---

**Group: review-system-design — HIGH**
Main conclusion: Reuse review artifacts across stages (62% time reduction, simulation N=25,000); target second pass at borderline outputs.

Counter-hypothesis: Artifact reuse introduces bias carryover — a borderline first-pass finding carries the first reviewer's framing into the second pass, which should be independent. The 62% time reduction comes at the cost of a structurally compromised independence guarantee.

Assessment: PLAUSIBLE as a usage condition. The synthesis's own Assumption 4 adversarial check (Step 4) already flagged this risk. The `cascade-peer-review-62pct-time-reduction` (HIGH) explicitly measured quality: Hellinger distance 0.452 vs. 0.433 with reuse — approximately 4.5% quality reduction. The simulation tested the reuse scenario and found quality held. Scope condition: reuse is safe where **independence is not the stated purpose of the second pass** (disagreement resolution, efficiency); unsafe when the second pass is designed to provide an independent verdict. Targeting second pass at borderline outputs (not all outputs) is consistent with this — it explicitly positions the second pass as resolution, not redundant independence-check.

**Verdict:** PLAUSIBLE with usage condition. No tier change. Main conclusion stands; scope qualifier confirmed by the simulation's own measurement.

---

**Group: alternatives-generation (counter-prior finding only) — HIGH**
Main conclusion: Frontier models achieve <15% hypothesis discovery rate on counter-prior cases even when explicitly tasked (HypoBench, all tested frontier models); synthesis is structurally biased toward confirming priors; forced counter-prior pass is the structural mitigation.

Counter-hypothesis: The <15% rate is benchmark-specific, not a general property — richer context about the field's known priors, or larger model capacity, might achieve substantially higher rates. Additionally, the forced pass generates plausible-sounding counter-hypotheses that remain prior-consistent anyway; it adds procedural overhead without genuine counter-prior coverage.

Assessment: PLAUSIBLE as known limitation — this is explicitly addressed in Step 5 GAP 3 of this synthesis: "forced pass may produce counter-sounding but prior-consistent outputs. External signal injection would be more effective." The <15% HDR finding is from a benchmark explicitly designed to test counter-prior hypothesis discovery at varying complexity — it is a measured outcome, not inferred. The counter-hypothesis about "richer context" is speculative; no study documents recovery to >50% with prompt enrichment on counter-prior tasks. The "adds overhead without coverage" concern is real but does not negate the marginal improvement — forced pass is a floor improvement (even prior-consistent counter-hypotheses force explicit acknowledgment), not a complete solution.

**Verdict:** PLAUSIBLE as scope limitation. No tier change. HIGH finding stands; floor vs. ceiling improvement distinction is logged as known limitation.

**Group: adversarial-review-dynamics — HIGH (structural requirements)**
Main conclusion: Adversarial review must be pre-commitment (before crystallized stakeholder position); reviewer and generator must be structurally separated (`red-team-late-timing-resistance` HIGH, qualitative N=17; `red-team-internal-familiarity-blindspot` HIGH).

Counter-hypothesis: Late adversarial review (after deliverable is developed and concrete) is more effective precisely because the target is specific enough to meaningfully attack — early adversarial review on vague proposals produces shallow, non-specific criticism. "Pre-commitment" as a timing requirement is overbroad: the real requirement is "concrete target," which may only exist late in development.

Assessment: PLAUSIBLE as a timing clarification rather than a contradiction. `red-team-well-developed-object-prerequisite` (CONSENSUS) explicitly states adversarial review requires a "crystallized target" — this IS the counter-hypothesis's premise. The synthesis's Step 2 contradiction scan (Pair 3) already resolved this: the optimal timing is **crystallized target + pre-stakeholder-commitment** (specific enough to attack, but before political investment in proceeding). The finding that late review gets suppressed (`red-team-late-timing-resistance`, HIGH, qualitative N=17) is about stakeholder resistance to late-stage criticism, not about criticisability — the target is concrete but the organization is committed. These are two different barriers. Counter-hypothesis correctly identifies "crystallized target" as required; fails to account for the suppression mechanism that fires after stakeholder commitment regardless of target specificity.

## Step 3.7 — Principled Derivation (2026-07-05)

### search-methodology-fundamentals (HIGH)
**Finding (data point):** 78.1% of search errors affect recall and only 21.9% affect precision (N=137 reviews), making false negatives the dominant failure mode — false positives are filterable at screening, false negatives are invisible. Iterative query evolution (berrypicking, Bates 1989, 3,187 citations) is required because pre-specified queries cannot anticipate vocabulary that only surfaces from retrieved results; each result generates new terms and sub-questions the original queries could not.

**Principle (Sub-step A):** When a retrieval task has asymmetric error costs — missing an item is unrecoverable after the search closes, but a retrieved false positive can be filtered at a later stage — and when the full vocabulary of what to look for is only partially knowable before retrieval begins (relevant terms and sub-questions only surface from what is retrieved), any pre-specified and fixed retrieval strategy will systematically underperform an adaptive strategy that treats each result as a source of new query terms and sub-questions. The cost structure demands recall-biased behavior; the vocabulary opacity demands iteration. These two conditions reinforce each other: the asymmetric cost raises the stakes of missed items, while vocabulary opacity guarantees that pre-specification will miss items from concept classes the searcher had not yet encountered.

**Fit to our setup (Sub-step B):** Tested in: systematic literature reviews — human experts searching bounded academic databases with partially controlled vocabularies, where the corpus is finite and the review protocol is fixed at registration. Kit-making setup differs in: (1) the "corpus" being searched in Phase 3 is the open web plus grey literature — less indexed, less bounded, vocabulary less controlled; (2) some searching is done by an LLM agent rather than a human expert, introducing model-level retrieval biases not present in human search; (3) the kit has a post-ship revision path (weakness_register.md, phase 7 audit) that partially reduces irreversibility of missed mechanisms, weakening the recall-asymmetry somewhat; (4) research angles must be generated before they can be searched, adding a meta-retrieval layer (finding what to search for) that the tested setup collapses into protocol design. The principle still applies because: mechanisms missed in Phase 3 cannot be introduced in Phase 5 without restarting research — the synthesis-to-plan pipeline has no backfill mechanism, so the functional irreversibility holds even if formal revision is possible. Vocabulary emergence is directly observable: kit-maker Phase 3 consistently produces RESEARCH_ANGLES.md growth mid-run because adjacent domains surface concepts Phase 2 did not name. Principle applies less strongly if: the domain is well-mapped from a prior kit iteration (prior synthesis.md exists and is referenced); or the kit's revision path is short enough that a missed mechanism can be caught in Phase 7 testing before the kit ships.

**Derivations beyond the finding (Sub-step C):**
- The Phase 3 research plan should treat angle-generation as an ongoing output, not a front-loaded input. The current procedure populates RESEARCH_ANGLES.md before execution begins; the principle implies it should also grow during execution — each completed research batch is required to yield candidate new angles before the next batch starts, rather than batch launches being driven only by the pre-specified list. This is a structural change: angle discovery is a product of Phase 3, not only a prerequisite. Confidence: high — the vocabulary-emergence mechanism is directly present in kit-maker Phase 3 as-executed.
- When the procedure must choose between breadth (more angle classes) and depth (more sources per angle), the asymmetric cost structure should bias it toward breadth until saturation evidence exists. Saturation is the observable signal that vocabulary emergence has stopped; until that signal is present, adding another angle class is lower-risk than stopping early. The current saturation criterion captures this implicitly; the principle implies it should be stated explicitly as the governing heuristic so it does not compete equally with depth-first alternatives during execution. Confidence: medium — the analogy between literature-recall and mechanism-recall holds mechanistically, but the saturation signal in open-web search is noisier than in database search.
- LLM agents executing the kit have the same asymmetry at runtime: a rule that fails to surface in the agent's context window cannot self-correct mid-task, but an over-retrieved rule can be ignored. This implies kit rules should be authored for high-recall retrieval by LLMs — redundant surface forms, multiple trigger phrasings, concrete observable conditions — rather than minimal-and-precise formulations that risk being absent at the moment they are needed. Precision in rule wording optimizes for the case where the rule is already retrieved; recall in rule surface form optimizes for whether the rule is retrieved at all. The current behavioral contract guidance does not distinguish these two axes of rule quality. Confidence: medium — assumes LLM context retrieval has recall-dominant error structure similar to keyword search, which is plausible but untested in kit-maker context.

### peer-review-quality (HIGH)
**Finding (data point):** Inter-reviewer agreement is r=0.34 (N=45-study meta-analysis), making a single-reviewer pass without rubric essentially random quality signal. 18% of psychology papers contain errors and 1-in-8 affect conclusions (`peer-review-psychology-18pct-errors`, HIGH). Domain-specialist sub-reviewers are the highest-ROI intervention; the scope condition is that multi-reviewer benefit is strongest for objective error detection (statistical, procedural, factual) and weaker for subjective quality dimensions where genuine expert disagreement is expected.

**Principle (Sub-step A):** When a single evaluator assesses complex output against an implicit or partially-specified standard, the variance in how that evaluator interprets the standard is indistinguishable from variance in the output quality being assessed. The resulting signal is unreliable. Reliability improves through two mechanisms that work independently: externalizing the standard into explicit, observable criteria (rubrics), and using multiple independent evaluators whose disagreement surfaces interpretation variance rather than absorbing it. The improvement from multiple evaluators concentrates on dimensions where an objectively correct answer exists; on dimensions where genuine expert disagreement is expected, adding evaluators increases inter-rater surfacing but does not converge the signal toward truth.

**Fit to our setup (Sub-step B):** Tested in: human domain-expert review of academic research papers, where evaluator variance comes from differing interpretations of implicit quality standards and domain knowledge gaps. Our setup (LLM agents auditing behavioral kits) differs in three ways: (1) the evaluator variance source is different — LLMs vary due to prompt sensitivity, context window position bias, and framing effects rather than cognitive or domain-expertise divergence; (2) the output being evaluated (kit rules and procedures) has a higher proportion of binary-observable criteria than research papers — many kit checks are objectively verifiable (required sections present, observable violation condition stated, format compliant), which narrows interpretation variance at the criterion level; (3) LLMs can be given identical prompts and context, reducing one source of inter-evaluator divergence that is unavoidable for human reviewers. The principle still applies because the structural problem — interpretation variance conflated with output quality signal — holds for LLMs; prompt-sensitivity and context effects produce the same epistemic failure mode. The principle applies most strongly to subjective kit dimensions (will this rule fire under execution pressure? is this rule over-specified to the point of brittleness?) and least strongly to binary criteria that leave minimal interpretation room. Net assessment: principle is applicable, but the severity of the reliability problem is lower than in the tested setup for binary-criterion passes, and higher for holistic quality passes — the two audit types should not be treated identically.

**Derivations beyond the finding (Sub-step C):**
- Audit criteria should be decomposed into binary observables before any audit pass runs. The reliability problem is smallest when each criterion is an unambiguous yes/no check and largest when the criterion is holistic. A pre-audit normalization step that converts all criteria to binary observables reduces interpretation variance structurally — without requiring multiple evaluators — and concentrates multi-evaluator investment on the dimensions where holistic judgment is unavoidable. Confidence in this derivation: high, because the mechanism (interpretation variance is minimized when interpretation space is minimized) holds directly from the principle and is consistent with the scope condition.
- A two-tier audit structure should separate objectively checkable criteria from subjective quality dimensions. One pass covers binary-observable checks (required sections, observable violation conditions, format compliance, section completeness); a second pass covers subjective criteria (pressure resistance, rule generalization scope, decay vulnerability). The principle implies multi-reviewer benefit concentrates on the second pass; the first pass can use a single evaluator with lower reliability risk if criteria are fully binary. Combining both into a single holistic pass wastes multi-reviewer resources on low-variance criteria while providing inadequate coverage for high-variance criteria. Confidence: medium, because the tier boundary (what counts as objectively checkable vs. genuinely subjective) requires explicit pre-specification and is itself a judgment call.
- Any kit certification verdict produced by a single audit pass should be treated as provisional, not final. The principle implies a single-pass audit cannot distinguish evaluator interpretation noise from real quality signal — the verdict requires at minimum a second independent pass (different agent instance, no access to first audit output) before it is treated as reliable for shipping decisions. This holds even when the single auditor is high-capability and follows a rubric, because the rubric reduces but does not eliminate interpretation variance at the criterion level. Confidence: medium-high; the derivation follows directly from the principle, but the practical cost of mandatory two-pass audits on every kit is non-trivial and the threshold for 'sufficient reliability' is not established by the research.

### review-system-design (HIGH)
**Finding (data point):** Reusing review artifacts across stages reduces reviewer effort by 62.9% (simulation N=25,000 researchers) with only approximately 4.5% quality reduction (Hellinger distance 0.452 vs. 0.433 without reuse); removing the second review pass entirely produces a 28% quality decline. The usage condition: reuse is safe where independence is not the stated purpose of the second pass (disagreement resolution, efficiency targeting borderline outputs); unsafe when the second pass is designed to provide an independent verdict.

**Principle (Sub-step A):** When a multi-pass evaluation process repeats prior work, the safety of carrying forward prior artifacts (decisions, annotations, partial results) into subsequent passes is determined entirely by the epistemic function the second pass must perform: passes designed to provide a clean-slate independent verdict are contaminated by prior context; passes designed to refine, continue, or target efficiency are neutral-to-improved by prior context. The distinction is not about quality of the artifact or effort level — it is about whether the pass's functional purpose requires freedom from prior framing to do its job.

**Fit to our setup (Sub-step B):** Tested in: simulation of human systematic reviewers (large N) making binary screening decisions across literature review stages. Our setup differs in: (1) the "reviewers" are LLM agents, not humans — the efficiency gain from reuse is not fatigue-driven but token-and-processing-driven, which changes the magnitude but not the directionality; (2) the artifacts being potentially reused are behavioral rules, procedure annotations, and synthesis text rather than screening decisions; (3) our pipeline is far smaller in volume, so effort-reduction effects are less dramatic in absolute terms. The principle still applies because the core mechanism — whether a pass's purpose requires a clean slate — is agent-type-neutral: it is a functional requirement of the evaluation design, not a property of human versus LLM cognition. It applies more strongly in the kit-making context where several passes already explicitly state independence as their purpose (Steps 3.5 and 3.6 subagents), making the distinction already load-bearing in the pipeline. It applies less strongly where the efficiency gain motivates the design (human-researcher fatigue has no LLM equivalent), so the principle should not be used to justify reuse primarily on effort-saving grounds — the safety condition is still the independence criterion, not the efficiency delta.

**Derivations beyond the finding (Sub-step C):**
- Every pass added to the kit pipeline should be classified at design time as 'independence-required' or 'continuation-acceptable,' with that classification stated explicitly in the pass specification — not left to the executing agent to infer. Unspecified passes will default to context-carrying (lower effort), which is safe only if independence is not the purpose. An ambiguous 'review again' instruction creates a safety condition the agent must infer rather than observe, and inference under execution pressure is documented to fail. Confidence: high — mechanism follows directly from the principle and the kit pipeline's existing documented failure modes around recognition checks.
- Auditor passes in Phase 7 whose purpose is 'catch what the builder missed' (continuation-acceptable) can legitimately receive prior audit context without undermining the audit's functional purpose; only passes explicitly designed to provide an independent verdict (independence-required) must be structurally isolated from prior synthesis and builder framing. This implies the Phase 7 audit specification should distinguish between these two pass types and enforce isolation only where the pass type requires it — not uniformly across all auditor activity. Confidence: medium — the principle supports the distinction, but the specific quality impact in the LLM-agent auditor context is unverified.
- Generated kit behavioral contracts that specify multi-pass review (e.g., iterative auditor rounds, peer-check procedures) should include an explicit declaration for each pass: does this pass require independence from prior pass output? If yes, the contract must specify what context is excluded and how exclusion is enforced. If no, prior context may be provided. Contracts that omit this declaration push a safety-critical inference onto the executing agent at runtime — the same class of failure the kit pipeline resolved by making independence structural in Steps 3.5 and 3.6. Confidence: high — this is a direct application of the principle to kit behavioral contract design requirements already in Hard Rule 13.

### alternatives-generation (HIGH (narrow: counter-prior finding) / CONSENSUS (TRIZ/lateral))
**Finding (data point):** All tested frontier models achieve less than 15% hypothesis discovery rate on counter-prior cases (HypoBench, arXiv:2504.11524v1), even when explicitly tasked with finding them — default synthesis is structurally biased toward confirming priors. This gap holds across all frontier models tested and at higher benchmark complexity. The forced counter-prior pass is a floor improvement (produces explicit acknowledgment); it does not reach ceiling improvement because models still weight priors heavily even under adversarial instruction — external signal injection (actual contradicting evidence) would be needed for full mitigation. TRIZ and lateral thinking claims in the same group are anecdote-level.

**Principle (Sub-step A):** A generative system shaped by accumulated pattern distributions cannot reliably surface outputs that contradict those distributions through instruction alone. Instruction shifts surface labeling — the system produces outputs with counter-prior framing — but the underlying generative weighting continues to draw from learned patterns. Genuine alternative generation requires not a different instruction but different input: material that makes the alternative content itself prominent, so generation is pulled toward it rather than repelled from a familiar prior. The gap between "instructed to find alternatives" and "actually finding alternatives" is closed by changing what the system reads before generating, not by changing what the system is told to do.

**Fit to our setup (Sub-step B):** The finding was tested in single-turn benchmark conditions where frontier models were directly asked to generate counter-prior hypotheses about research claims — the task was hypothesis discovery under explicit adversarial instruction. Kit-making differs in three structurally relevant ways. First, the agent is designing procedures for other agents to follow, not directly generating hypotheses — prior-anchoring operates at the procedural design level (which mechanisms get proposed, which structural patterns get baked into procedures) rather than at the claim level. Second, kit construction spans multiple phases and sessions, so early-phase design decisions accumulate into a session prior that weights all subsequent design; this is compounding rather than single-shot, meaning the anchoring effect grows across phases rather than staying constant. Third, and most importantly, the kit-maker setup provides structural control over what input material the kit-maker sees before generating — the benchmark setting had no such lever. This means the principle applies at least as strongly in kit-making (and likely more so given session accumulation), but the actionable control point is different: the lever is what the kit-maker reads before generating a mechanism, not what instruction the kit-maker receives. The principle's core mechanism — injection of contradicting material as the operative mitigation — is more feasible to implement structurally in kit-making than in single-turn benchmark conditions, because we control the procedure that precedes generation.

**Derivations beyond the finding (Sub-step C):**
- Mechanism brainstorming should require the kit-maker to read a contradicting design example — a kit or procedure that solved the same problem via a structurally different approach — as actual injected source material BEFORE generating candidate mechanisms, not as an instruction appended to the prompt. Without this injection, the mechanism list will overrepresent the most recently seen pattern family regardless of how the instruction frames the request for diversity. Confidence: medium — the injection mechanism is directly implied by the principle, but the delta in mechanism diversity from adding it has not been measured in this context.
- The TASK_TEMPLATE 'generate at least one alternative with a different underlying assumption' rule is likely producing surface variants within the same pattern family rather than genuine structural alternatives. To satisfy the intent rather than the letter, the rule should require the agent to name the assumption class the alternative departs from AND verify that the primary option and the alternative would produce incompatible recommendations on at least one concrete design question — not just different phrasings of compatible recommendations. An alternative that agrees with the primary option on every consequential design choice is not an alternative in the principle's sense. Confidence: medium — implied by the distinction between surface labeling and generative weighting.
- Inter-phase injection gates: at each phase transition, before the kit-maker proceeds to the next phase, inject at least one design example that directly contradicts the dominant structural choice made in the current phase. This prevents session-cumulative prior hardening, where phase N's output silently becomes the unquestioned anchor for phase N+1, and the kit converges on a single pattern family even though adversarial alternatives were required at each individual step. The injection does not have to be a full kit — a one-paragraph description of a different structural approach is sufficient to activate the mitigation mechanism. Confidence: low-medium — the inter-phase compounding dynamic is implied by the principle but has not been directly tested in procedural design contexts.

### adversarial-review-dynamics (HIGH (structural requirements) / CONSENSUS (mechanism))
**Finding (data point):** Late adversarial review (after stakeholder commitment to a deliverable) generates resistance that suppresses its findings regardless of target specificity (`red-team-late-timing-resistance`, HIGH, qualitative N=17). Reviewers internal to the team developing the object share familiarity blindspots that narrow the hypothesis space (`red-team-internal-familiarity-blindspot`, HIGH). The structural requirements are therefore: adversarial review must occur at crystallized-target AND pre-stakeholder-commitment timing (not either condition alone), and reviewer and generator must be structurally separated.

**Principle (Sub-step A):** A critique or review function's effectiveness is not intrinsic to the quality of what it finds — it depends on two jointly necessary structural conditions being present: (1) the review must occur before commitment to the reviewed object solidifies resistance to revision, and (2) the reviewer must operate from outside the cognitive or institutional frame that produced the object. Neither condition is sufficient alone. Timely review by an insider shares the generator's frame and produces critique within the same hypothesis space that generated the problem. Independent review after commitment encounters resistance that discounts findings regardless of their validity. The two conditions are a conjunction, not a disjunction — and because both must hold simultaneously, a review process that controls for one while neglecting the other is not adversarial review; it is adversarial-review-shaped process that will systematically fail to influence outcomes.

**Fit to our setup (Sub-step B):** Tested in: human organizational red-teaming of security or product deliverables, where stakeholder commitment is social and institutional — external parties (clients, management, sponsors) have bought into the deliverable, creating political resistance to revision. Reviewer familiarity blindspot was assessed in human teams sharing project history. Our setup differs in three ways: First, the commitment mechanism is architectural rather than social. In kit-making, commitment solidifies not through external stakeholder buy-in but through propagation — a design decision in Phase 5 PLAN cascades into Phase 6 files; once built, every file implementing the decision must be rebuilt to correct it. Resistance to late findings is cost-resistance, not face-saving. The timing condition still applies but the reason it applies is different. Second, the independence condition is harder to satisfy. In human teams, structural separation means different people or different organizations. In LLM kit-making, reviewer and generator may both be LLM agents sharing the same training distribution — which means they can share systematic hypothesis-space blindspots even when formally separate. Structural independence does not guarantee cognitive independence when base-model biases are shared. The familiarity blindspot finding applies, but the mechanism runs deeper than organizational familiarity. Third, the "crystallized target" timing landmark differs. In the tested setup, crystallization happens when external stakeholders sign off. In kit-making, the analogous landmark is when Phase 5 PLAN is drafted and accepted — the kit's structure is specific enough to review but Phase 6 build has not yet propagated the decisions. Principle applies most strongly before Phase 6 starts; applying it after Phase 6 is the exact failure mode the timing condition names, now expressed as correction-cost multiplication rather than social resistance.

**Derivations beyond the finding (Sub-step C):**
- Adversarial review of a Phase 5 PLAN must be a hard phase gate before Phase 6 build starts, not an audit run after Phase 6 is complete. The timing condition in the principle implies that once Phase 6 files implement a design decision, the correction cost commits the project to the decision in exactly the same way social commitment commits human stakeholders — the review may still find the flaw, but acting on it requires rebuilding every downstream artifact. The research tested social commitment; the mechanism extends to technical propagation commitment. Derivable rule: no Phase 6 file is written until an adversarial subagent has reviewed the Phase 5 PLAN and its verdict has been processed. Confidence in this derivation: high — the mechanism (commitment creates resistance to acting on findings) is the same; only the substrate (social vs. technical) differs.
- The adversarial review brief must be authored without the design rationale that produced the object under review. The familiarity blindspot finding shows that shared cognitive frame narrows the hypothesis space the reviewer can access. In LLM kit-making, if the main agent both designs the procedure and writes the adversarial review instructions, the review brief is produced by the same frame as the object — the adversarial reviewer inherits the generator's framing through the brief, even when structurally separate. Derivable rule: the adversarial reviewer receives the object (the PLAN or procedure text) directly, without the reasoning that produced it. Design rationale, rejected alternatives, and synthesis prose are withheld from the review brief. The reviewer must construct counter-arguments from the object itself, not from the generator's account of how the object was built. Confidence in this derivation: medium — the mechanism (shared frame narrows hypothesis space) is well-grounded; the specific pathway (brief authorship as frame transmission) is inferred, not directly tested.
- Pre-crystallization review is symmetric with post-commitment review in its ineffectiveness, implying a narrow optimal review window. The finding establishes that post-commitment timing suppresses findings via resistance. The principle implies a symmetric problem at the earlier end: review before the target crystallizes addresses a moving object — findings may be rendered irrelevant by subsequent design decisions, and the reviewer cannot generate counter-hypotheses against a target that has not yet stabilized. This means adversarial review is only effective inside a specific window: after the design is specific enough to hold under scrutiny but before implementation commits resources to it. In kit-making terms: after Phase 4 synthesis is complete and Phase 5 PLAN is drafted (crystallized target), but before Phase 6 build starts (pre-commitment). Reviewing rough notes in Phase 3 is too early; reviewing finished Phase 6 files is too late. Derivable rule: adversarial review is phase-gated to the Phase 4-complete / Phase 5-final / Phase 6-not-started window. Confidence in this derivation: medium — the post-commitment side is directly supported; the pre-crystallization side is derived by symmetry of mechanism, not from direct evidence.

### query-formulation-quality (CONSENSUS)
**Finding (data point):** Expert searchers diagnose the failure mode and switch strategy class when a query fails (reformulating radically — changing vocabulary category, index, or framing); novices repeat variants of the same query (`expert-strategy-switching-on-failure`, MEDIUM). Domain-independent search training is the finding for the librarian-coauthor effect but rests on n=10 (small sample, HIGH-tier claim not supported). Strategy-switching and structured query design are consensus-level findings with mixed empirical support.

**Principle (Sub-step A):** When an approach fails, the failure is diagnostic information about strategy class mismatch — not a signal to adjust parameters within the same approach. Effective agents identify what the failure reveals about the problem's structure and select a categorically different approach; ineffective agents vary surface parameters (vocabulary, phrasing, order) while leaving the underlying strategy class unchanged. The gap between these behaviors is not effort or knowledge — it is whether the agent treats failure as feedback about kind rather than degree.

**Fit to our setup (Sub-step B):** The finding was tested with human searchers doing information retrieval tasks — agents with tacit metacognitive awareness who could observe their own failure and self-direct a strategy switch. Our setup (behavioral kits for LLM agents) differs in two critical ways: (1) LLM agents do not have spontaneous metacognitive monitoring of failure type — they will not self-diagnose "this is a strategy-class mismatch" unless the kit structures that diagnostic step explicitly; (2) LLM agents process a task in one pass unless the kit encodes an iterative retry loop with explicit failure classification gates. This means the principle applies, but only indirectly — it cannot be implemented as an agent disposition ("be like an expert searcher") but must be implemented as kit structure (explicit failure-type taxonomy, branching logic per failure class, diagnostic checkpoint at each retry). The principle applies more strongly when the kit operates on tasks with multiple recoverable failure modes and structured output evaluation; it applies less strongly when the task is one-shot with no retry surface, or when all failure modes map to the same recovery action.

**Derivations beyond the finding (Sub-step C):**
- Kit retry loops must encode a failure-type taxonomy, not just a retry trigger. A rule that says 'if output is poor, try again' selects no recovery action — it defaults to parameter variation within the same strategy. The kit needs to distinguish failure classes (e.g., 'answer addresses wrong question' vs. 'answer is correctly framed but wrong in content' vs. 'answer is too abstract to act on') and map each class to a different recovery strategy, not a rephrasing of the same prompt.
- Phase-gate self-checks in the kit should not ask 'is this output good enough?' but 'what failure type does this output exhibit, if any?' — because 'not good enough' is a degree judgment that implies try harder, while a failure-type diagnosis implies switch approach. The check question is the mechanism: the wrong question installs the novice pattern, the right question installs the expert pattern.
- When a kit mechanism produces poor results across multiple runs, the diagnostic protocol should explicitly branch on 'misconfigured mechanism' (parameter issue — same strategy class, different settings) vs. 'wrong mechanism in kind' (strategy mismatch — requires selecting a different mechanism type). Without this branch, the default repair action is always 'add more instructions to the same mechanism,' which is the documented novice failure mode applied at the kit-design level.

### automated-query-optimization (CONSENSUS (with 1 HIGH))
**Finding (data point):** Recall and precision are separate bottlenecks in retrieval pipelines requiring separate mechanisms (`rag-recall-precision-bottleneck`, HIGH): FlashRank achieves 5.4% NDCG gain, 6-8% accuracy gain, and 35% token reduction by separating retrieve-for-recall from rerank-for-precision as distinct pipeline stages. Dual-query expansion adds approximately 6.7% recall on QA benchmarks (`query-expansion-recall-improvement-6-7-percent`, MEDIUM) but also introduces precision dilution on ambiguous terms — the architectural solution is expand first, filter later.

**Principle (Sub-step A):** When an output pipeline requires two properties that trade off against each other, a single mechanism optimizing for both simultaneously performs worse on each than two sequential dedicated stages — one that maximizes coverage, one that enforces constraints. The ordering is load-bearing: the expansive stage must precede the filtering stage; reversing the order compounds the failure modes of each, because a filtering mechanism operating on an already-narrowed candidate set cannot recover coverage, while an expansion mechanism operating after filtering adds noise without correcting the prior loss.

**Fit to our setup (Sub-step B):** Tested in: dense vector retrieval pipelines on QA benchmarks, with document chunks as the unit being retrieved and ranked, and NDCG / accuracy as the metrics. Our setup differs in that the "pipeline" is behavioral kit construction for LLM agents — the units being generated are rules, procedures, and mechanisms (not document chunks), and the quality dimensions are coverage (does the kit handle the full space of situations the agent will face?) versus actionability/precision (are the rules specific enough to fire correctly without drift?). The principle still applies because the underlying tension is structural: a rule-writing agent instructed to produce comprehensive AND high-quality rules simultaneously will anchor on the quality filter and undergenerate candidates, or anchor on comprehensiveness and overgenerate noise — the same recall/precision tradeoff in a different medium. The principle applies more strongly when the kit's problem space is large and underspecified (the agent cannot enumerate the full situation space while also evaluating rule quality). It applies less strongly when the kit is narrow and the situation space is small enough that a single-pass review can cover it without systematic omission.

**Derivations beyond the finding (Sub-step C):**
- Kit rule generation should be a two-pass architecture: a breadth pass that surfaces all candidate rules for the target domain without quality filtering (capturing the full situation space), followed by a dedicated culling pass that applies specificity, actionability, and conflict checks. Single-pass rule generation that tries to be both comprehensive and high-quality will systematically sacrifice coverage — the filtering criterion dominates during generation and the agent never produces the candidates it would have filtered in anyway.
- Research angle generation (Phase 3) should separate angle enumeration from angle prioritization into explicit sequential steps rather than producing only angles that already seem high-relevance. An agent instructed to produce 'the most relevant angles' applies a precision filter during generation and underproduces coverage — exactly the recall failure mode. The enumeration-expansion rule in kit-maker CLAUDE.md is a partial instantiation of this principle, but the underlying mechanism (dedicate one stage to breadth, dedicate a separate stage to filtering) is not named, which means the rule can degrade to 'produce more than the user's examples' without enforcing actual architectural separation.
- Agent intake question selection should be two-stage: first surface all candidate unknowns without applying the LHAW filter (outcome-critical / action-counterfactual / irreversibility-gated), then apply the filter to the candidate set. Single-stage 'only ask essential questions' instructions conflate generation with evaluation and cause systematic underfiring — the agent applies the filter during generation and never produces the questions it would have evaluated and correctly retained. The structural gate required in Hard Rule 13 element 8 addresses the sequencing but does not enforce the two-stage generation architecture.

### adversarial-llm-specific (CONSENSUS)
**Finding (data point):** Default prompting produces 77% success prediction at 22% actual success rate; adversarial framing ('try to refute this claim') improves calibration (`adversarial-framing-calibration-gain`, MEDIUM, arXiv:2602.06948). LLM reviewers can be gamed via paraphrase (`llm-reviewer-gaming-via-paraphrase`, MEDIUM, confirmed). Adversarial review completion ranges from 0% to 80% in autonomous mode versus 100% with structured role decomposition (planning, execution, judgment as separate roles) (`expert-scaffolding-vs-autonomous-gap`, MEDIUM).

**Principle (Sub-step A):** When an LLM agent is asked to generate and evaluate within the same cognitive frame, its evaluative output drifts toward confirming its generative output — a structural artifact of role conflation rather than a failure of intent. Separating the generative role from the evaluative role, or injecting a committed adversarial stance that is structurally incompatible with the confirming role, is necessary to break the confirmatory loop. This applies whether the conflation is prompt-level (same instruction producing both generation and evaluation), agent-level (same agent doing both), or phase-level (same session context spanning both). Surface variation in the content being evaluated does not break the confirmatory loop — only structural role separation does.

**Fit to our setup (Sub-step B):** Tested in: single-turn or short-sequence LLM evaluation tasks — claim verification, autonomous review, calibration against ground truth. Kit-making setup differs in three ways. First, the generative and evaluative roles are conflated across multiple phases rather than a single turn — the same agent writes procedures in Phase 6 and assesses compliance in Phase 7, meaning confirmatory drift compounds across phases rather than occurring once. Second, the surface area for paraphrase gaming is larger: kit rules are written in natural language prose with inherent ambiguity, so a compliance check by the authoring agent has more interpretive latitude to find satisfaction. Third, kit-making already has partial structural role separation (phase-output gates, subagent dispatches) — meaning the principle is partially instantiated, which means it applies more strongly in the gaps where role separation is absent (notably: done-criteria verification by the executing agent, and behavioral-contract self-audit). The principle applies more strongly here than in the tested setup because multi-phase compounding increases total confirmatory drift, and because the stakes of a missed violation in a behavioral contract are higher than in a single evaluation task. The principle may apply less strongly for Phase 3 research claims, which are checked by verify passes that use external sources as the ground truth — that is genuine structural separation already present.

**Derivations beyond the finding (Sub-step C):**
- Self-evaluation of done-criteria by the executing agent is structurally unreliable — not because agents are careless, but because the generative-evaluative role conflation is maximal at task close (the agent that executed is the agent assessing whether execution met the criteria). This implies done-criteria gates must be evaluated against observable artifacts with binary presence (file exists, field is filled, count matches) rather than qualitative self-assessment. Confidence in this derivation: high — the mechanism is the same role-conflation dynamic from the finding, applied to the specific moment of done-criteria checking.
- Kit behavioral contracts written in NEVER-rule (prohibition) form are more gameable by the agents running them than commission-form rules specifying a positive action under a named condition. Prohibition is a frame over all possible actions — an agent checking compliance with a NEVER-rule can find any paraphrase of the prohibited action acceptable, because the evaluative frame is open-ended. Commission-form rules ('do X when condition Y is observable') are harder to paraphrase around because the positive action is either present or absent in the output. This implies that rules in generated kits should default to commission form, with NEVER-rules converted to their commission equivalents where feasible. Confidence: medium — the mechanism is sound but the paraphrase-gaming finding was demonstrated in claim-review, not behavioral-contract compliance; the transfer requires that the same surface-variation sensitivity applies to rule interpretation, which is plausible but untested.
- The 0-to-100% completion gap between autonomous and role-decomposed scaffolding implies that self-reported completion rates are not a reliable signal of task quality when the same agent completes and certifies. This extends to kit audit procedures: a Phase 7 audit where the authoring agent runs the audit against its own Phase 6 output will systematically find fewer violations than an audit by a structurally separated reviewer. This implies Phase 7 audit dispatch must go to a subagent with no session context from the authoring phases — not as an optimization but as a structural requirement, because the authoring agent cannot exit its confirmatory frame regardless of explicit instruction to be critical. Confidence: medium-high — the mechanism maps directly, but the 0-to-100% gap was measured in a specific task type (autonomous review) and the exact gap magnitude in kit audit is unknown.

### structured-decision (CONSENSUS)
**Finding (data point):** 9 of 9 MCDA frameworks surveyed do not guide the actual decision (`mcda-nine-frameworks-lack-guidance`, consensus); quantified rankings obscure the value-laden weighting choices rather than making them explicit — frameworks that appear to decide actually suppress deliberation (`mcda-should-support-not-supplant-deliberation`, consensus). No HIGH-confidence empirical basis exists in this group; both findings are standard MCDA critique literature. Action consequence: always surface weights explicitly and always include a recommended action alongside the framework output.

**Principle (Sub-step A):** When a structured tool converts value-laden judgments into a mechanical output — a score, ranking, or verdict — without surfacing the values embedded in its construction, it transfers the appearance of objectivity to what is actually a choice. The output looks like a conclusion but is a reframing of the question: the deliberation required to reach a decision has not been performed, it has been suppressed. A tool that appears to decide is not neutral — it is one that has pre-decided on behalf of the user without disclosure. The corrective is not to add more structure; it is to make the embedded values visible and to pair any structured output with an explicit recommended action, keeping the value judgment and the action recommendation as two separate, inspectable artifacts.

**Fit to our setup (Sub-step B):** Tested in: MCDA literature — multi-criteria decision analysis tools used by human planners and policy analysts facing complex tradeoffs (infrastructure, procurement, policy). The critique operates at the level of human decision-support: humans fail to notice that quantified rankings encode their own weighting assumptions and mistake a re-presentation of their values for an external answer.

Our setup differs in three ways. First, the agent is not a human decision-maker — it has even less metacognitive access to the framework it is executing. A human MCDA user may notice that the weighting step was arbitrary; an LLM agent following a kit procedure that produces a score will treat the score as authoritative because it cannot inspect the rubric's implicit assumptions without explicit scaffolding to do so. This makes the suppression-of-deliberation failure mode more severe in our context, not less.

Second, the mechanisms at risk are different. In MCDA, the hidden-value problem occurs in the weighting step. In kit procedures, the analogous failure occurs wherever a mechanism delivers a verdict, ranking, or recommendation without naming what it is optimizing for — goal checkers, auditors, alternatives rankers, and reviewer outputs all carry this risk.

Third, kit procedures can be designed to expose their criteria structurally; this is a design choice available to us that MCDA frameworks often do not make. The principle applies more strongly in our context for the first two reasons and is more tractable to fix for the third. It applies less strongly only when a kit mechanism already names its optimization target explicitly alongside its output — in that case the suppression is not occurring.

**Derivations beyond the finding (Sub-step C):**
- Any kit evaluation mechanism — goal checker, auditor, code reviewer — that delivers a verdict (pass/fail, GREEN/YELLOW/RED, block/proceed) without exposing the criteria behind the verdict is applying the same suppression. The verdict replaces deliberation rather than informing it. Derived rule: every evaluation output in a generated kit must name its evaluation criteria alongside the verdict as two separate, inspectable fields. A verdict field without a criteria field is an incomplete evaluation output. Confidence in this derivation: high — the mechanism is identical to the MCDA case; only the domain differs.
- An alternatives-generation step that presents options as a ranked list without showing the dimension on which each alternative wins encodes a value preference (what counts as better) without disclosure. The user sees rankings but cannot reject the ranking by rejecting its assumption — they can only accept or refuse the output wholesale. Derived rule: any alternatives list in a kit procedure must expose the winning dimension for each alternative (the axis on which that option would be the correct choice), not just its rank. This matches the already-documented alternatives-depth principle but grounds it in the suppression mechanism rather than in effort norms alone. Confidence: high — directly derivable from the principle's mechanism.
- A kit rule that recommends a course of action without stating the value-assumption it is optimizing for makes it impossible for the user to accept the recommendation on one occasion and reject it on another based on which assumption applies. The recommendation becomes monolithic. Derived rule: every recommended action in a kit procedure should be paired with the assumption it presupposes (e.g., 'Recommended: chunk the input — assumes token budget is the primary constraint; override if quality-per-pass is the binding constraint'). This turns recommendations from verdicts into conditional guidance the user can evaluate against their current context. Confidence: medium — the mechanism holds, but the implementation overhead per rule is non-trivial and must be weighed against rule density limits already flagged in the behavioral contract.

**PLAUSIBLE WITH SCOPE CAVEAT:**
- search-methodology-fundamentals: HIGH applies where exhaustive recall is the protocol-specified goal (systematic reviews, Cochrane-style evidence synthesis). For general research contexts where precision/utility trade-offs govern source selection, the evidence base does not support the "required" framing.

The designation holds only for unstructured review contexts where rubric-based instruments are unavailable or impractical.
- review-system-design: The "optimal design" recommendation combines evidence from two separately tested simulation conditions (reuse vs. no-reuse; second-pass vs. no-second-pass) rather than directly testing the proposed combined design. The 62.9% effort reduction and 4.5% quality loss figures apply to the reuse condition alone; whether those figures hold when reuse is paired with targeted second-pass review of borderline outputs is not established by the evidence. Additionally, Hellinger distance as the quality proxy captures distributional similarity to a reference distribution, which may systematically underestimate quality loss in domains where catching specific high-stakes errors matters more than overall distributional alignment.
- adversarial-review-dynamics: The pre-commitment timing condition is necessary only when the reviewer lacks externally-enforced independence of interests; in adversarial review structures with structural incentive separation (opposing counsel, independent external auditors, short-sellers, post-mortem facilitators with no stake in the outcome), late timing does not reliably produce suppression and the timing condition is not jointly necessary.

**CONTESTED — reclassified from HIGH:**
- **alternatives-generation**: Three specific arguments substantially undermine HIGH. First, the model set tested (GPT-4o-mini, Qwen-2.5-72B, Llama-3.1-70B, DeepSeek-R1-Distilled-Llama-70B) excludes every flagship frontier model from the same period: GPT-4o full, Claude 3.5/3.7 Sonnet and Opus, Gemini 1.5/2.0 Pro, and o1/o3 extended-thinking models. Second, the <15% HDR applies only at the intersection of counterintuitive ground truth AND high compositional complexity (depth 3-4). The paper's own data shows that under counterintuitive + low complexity, most models recovered ground-truth hypotheses adequately; the severity is conjunctive, not unconditional. The KEY CLAIM strips this interaction and presents a tail condition as a general structural property. Third, "regardless of adversarial instruction" is unsubstantiated — the paper reports no variation in adversarial instruction format, so the claim extends well beyond what was empirically tested. These three points together mean the HIGH claim rests on a mischaracterized model set, a condition-stripped framing of a tail result, and an untested universality qualifier.
