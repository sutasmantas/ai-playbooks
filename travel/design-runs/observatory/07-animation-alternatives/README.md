# Observatory Animation Alternatives

Prepared: 2026-06-02

This folder is the second-pass alternatives pack based on the larger CSS/animation source pools.

Use it when Claude Design wants a different option from the core `06-code-snippets/` pack.

## Alternatives

| File | Use For | Best Source Pool |
|---|---|---|
| `svg-mask-cosmos-transition.html` | Section/page transition from road/warm layer to cosmos layer | Codrops SVG mask transitions with GSAP/ScrollTrigger |
| `responsive-curved-orbit-route.html` | Responsive orbit/route path that adapts to screen size | Codrops responsive curved path animations with GSAP |
| `css-particle-sky-background.html` | Lightweight hero/background star particles without Three.js | FreeFrontend CSS particle backgrounds / CodePen-style particle systems |
| `sparkle-orbit-cta.html` | More magical CTA/button hover alternative | FreeFrontend sparkle button / UIverse space buttons |
| `futuristic-weather-state-button.html` | Booking/weather status button with state feedback | FreeFrontend animated futuristic state button |
| `sci-fi-loader-alternative.html` | Loader alternative if planetary loader is too soft | FreeFrontend sci-fi loader / UIverse loader pools |

## When To Use

- Use these only if the core snippets feel too plain or too locked-in.
- Do not layer all of them into the final site.
- Prefer Codrops/GSAP alternatives for signature mechanics.
- Prefer FreeFrontend/UIverse-style alternatives for small UI states and CTA polish.

## Source Priority

1. Codrops + GSAP: signature transitions, SVG masks, route/orbit motion.
2. FreeFrontend + CodePen + UIverse: particles, buttons, loaders, small UI states.
3. Awwwards/Godly: quality benchmark and mood, not code.
