Adversarial subagent (Q3b — constructed independently from confirmatory results):
Claim challenged: "Decomposing a vague one-sentence research prompt into sub-questions improves domain coverage."
Counter-case: The weakest assumption is that the decomposer possesses the domain knowledge required to identify which sub-questions are needed. For truly unfamiliar domains, the LLM generates sub-questions reflecting training data coverage, not the actual research territory. Decomposition works as a recall mechanism for known-unknowns — questions the LLM already knows exist. Unknown-unknowns (gaps not reflected in training data, cross-disciplinary connections, novel research angles) are structurally absent from LLM-generated decompositions. The benchmark validation evidence uses HotpotQA and MuSiQue — datasets with ground-truth multi-hop paths the LLM has likely seen in training. These measure "does decomposition help organize known knowledge" not "does decomposition surface unknown territory." The condition where the original claim breaks: vague prompt in a novel domain the LLM's training corpus doesn't cover well. Under those conditions, decomposition produces confident-looking but systematically incomplete sub-question lists. The practitioner source (Caulfield) inverts the prescription: "Search for the known unknowns, LLMs for the unknown unknowns" — LLMs surface what's in the linguistic vicinity, not what's absent from their training.

# A01 — question-decomposition-sparse-input

## Key findings

### [key: structured-decomposition-frameworks-domain-coverage]
- evidence: consensus
- confidence: HIGH
- group: systematic-decomposition
- sources: https://library-guides.ucl.ac.uk/systematic-searching/research-question-frameworks, https://pmc.ncbi.nlm.nih.gov/articles/PMC4310146/

**Claim:** Systematic review methodology has developed 14+ explicit frameworks (PICO, SPIDER, CIMO, SPICE, PECO, EPICOT, etc.) that decompose a research question into named structural dimensions. Each dimension represents a category of information that must be specified for complete domain coverage. PICO covers Population / Intervention / Comparison / Outcome; SPIDER covers Sample / Phenomenon of Interest / Design / Evaluation / Research Type; CIMO covers Context / Intervention / Mechanism / Outcome. The consensus across decades of systematic review practice is that unstructured keyword searching fails to achieve reproducible coverage; forcing explicit enumeration of each dimension converts a vague research goal into a structured search strategy. The mechanism: named dimensions act as a checklist of what must be specified — when a dimension is blank, it signals an unresolved information need rather than an absence of requirement.

**Practical implication:** A kit procedure for handling vague research prompts should include a framework-selection step that maps the research type to the appropriate decomposition template (intervention question → PICO; qualitative/experience question → SPIDER; mechanism question → CIMO), then prompts the user to fill in each dimension. Blank dimensions are intake questions, not optional.

### [key: ltm-decomposed-prompting-organizational-not-knowledge]
- evidence: empirical
- confidence: MEDIUM
- group: llm-decomposition-mechanics
- sources: https://arxiv.org/abs/2205.10625, https://arxiv.org/abs/2210.02406, https://arxiv.org/abs/2602.04853

**Claim:** Least-to-Most Prompting (Zhou et al., 2022, arXiv:2205.10625) and Decomposed Prompting (Khot et al., 2022, arXiv:2210.02406) demonstrate that LLMs can generate valid sub-questions from a complex prompt and solve them sequentially, with large accuracy gains on compositional benchmarks (GPT-3 achieving 99% vs Chain-of-Thought's 16% on SCAN). However, the critical scope condition from arXiv:2602.04853 (2026): decomposition helps when the LLM already has the factual knowledge but lacks organizational structure to apply it. When the LLM lacks the underlying knowledge, decomposition does not help — and in fact, frontier models show diminishing accuracy gains from decomposition because they are already organizationally capable. The mechanism is not knowledge discovery — it is knowledge organization. Decomposition generates sub-questions in the LLM's vocabulary about the LLM's training domain; it does not surface what is absent.

**Falsification:** This mechanism-claim (decomposition = organization, not knowledge) would be falsified if a study showed LLM decomposition consistently generating sub-questions identifying knowledge gaps the LLM cannot answer and routing them correctly to external sources — demonstrating detection of one's own unknowns rather than organization of existing knowledge.

**Verify note:** (blank — empirical, verify pass should run; n-size concern: SCAN benchmark has clear compositional structure; real-world research prompts may differ substantially from compositional generalization tasks)

### [key: llm-query-expansion-fails-ambiguous]
- evidence: empirical
- confidence: HIGH
- group: llm-decomposition-failure-modes
- sources: https://arxiv.org/abs/2505.12694

**Claim:** arXiv:2505.12694 (accepted SIGIR 2025) finds that LLM-based query expansion significantly degrades retrieval effectiveness when (a) the LLM lacks knowledge of the query topic, or (b) query ambiguity is high. In the high-ambiguity case, the LLM produces "biased refinements that narrow search coverage" — it resolves the ambiguity by committing to one interpretation and generates expansions that all follow from that interpretation, eliminating alternative valid interpretations. The mechanism: ambiguity-resolution via training-data priors is the default LLM behavior; the LLM does not recognize it is disambiguating because it does not represent the alternative interpretations. Scope condition: expansion helps for low-ambiguity, well-represented queries; expansion actively hurts for high-ambiguity, underrepresented queries — exactly the cases that prompt decomposition is supposed to address.

**Falsification:** This failure mode would be falsified if a study showed that explicitly prompting the LLM to enumerate multiple interpretations before expanding (rather than expanding directly) eliminates the narrowing effect for ambiguous queries.

**Verify note:** (blank — empirical, SIGIR 2025 venue is high-quality; single study, replication needed)

### [key: underspecification-llm-infers-41pct]
- evidence: empirical
- confidence: MEDIUM
- group: underspecification-magnitude
- sources: https://arxiv.org/abs/2505.13360

**Claim:** arXiv:2505.13360 finds that LLMs infer unstated requirements correctly only 41.1% of the time from underspecified prompts — meaning that when a prompt omits a requirement, the LLM satisfies it about 41% of the time by defaulting to reasonable inferences. The remaining 59% of missing requirements produce degraded or incorrect output. Additionally, underspecified prompts are 2x as likely to regress (accuracy decreasing by >20%) across model updates or prompt changes, making them structurally fragile. The authors found that standard prompt optimization approaches provide minimal improvement, and propose systematic requirements discovery + evaluation + monitoring as the structural fix. Scope: this is a general finding across LLM task types, not specific to research question decomposition.

**Falsification:** This 41.1% figure would be falsified if a study using the same requirement specification approach showed substantially higher correct-inference rates on a different task distribution, suggesting the finding is task-specific rather than general.

**Verify note:** (blank — empirical, verify pass should confirm the 41.1% number from the actual paper)

### [key: reference-interview-iterative-probing-model]
- evidence: consensus
- confidence: HIGH
- group: information-elicitation-procedure
- sources: https://en.wikipedia.org/wiki/Reference_interview, https://pmc.ncbi.nlm.nih.gov/articles/PMC35232/, https://www.researchgate.net/publication/27403092_The_Reference_Interview_Theories_and_Practice

**Claim:** The reference interview — a 50+ year tradition in library and information science — is the validated procedural model for converting a vague patron query into a structured, actionable information need. Core principles: (1) initial user statements are treated as provisional and often misleading — the stated query is not the actual information need; (2) the interview proceeds through iterative probing and clarification to surface latent goals, unstated constraints, and hidden assumptions; (3) it is a collaborative meaning-making activity where both parties jointly refine the problem definition. The reference interview treats disambiguation of "airways" (aviation vs. medicine) as a structural requirement, not an optional nicety. The mechanism: explicit probing externalizes the hidden problem structure that users cannot articulate directly because they lack the vocabulary to describe what they need precisely until they see candidate structures.

**Practical implication:** A kit intake procedure should model the reference interview: treat the initial prompt as provisional input; apply a small structured set of clarifying probes (scope, output type, use case, what partial answers already exist, what would make the answer useful vs. not useful); do not proceed to decomposition until the probing pass resolves these.

### [key: unknown-unknowns-decomposition-ceiling]
- evidence: consensus
- confidence: HIGH
- group: llm-decomposition-failure-modes
- sources: https://mikecaulfield.substack.com/p/search-for-the-known-unknowns-llms, https://aclanthology.org/2024.findings-acl.383/

**Claim:** A structural ceiling exists for LLM-based question decomposition: decomposition surfaces known-unknowns (gaps the LLM's training knows exist and knows how to phrase) but cannot surface unknown-unknowns (gaps that are absent from the training distribution — cross-disciplinary connections, novel research angles, questions that only become visible after reading the literature). The ACL 2024 paper on "Knowledge of Knowledge" (arXiv:2305.13712) documents that LLMs confabulate answers to cross-domain questions requiring integration from two fields absent from training data. The practitioner formulation (Caulfield): "Search for the known unknowns, LLMs for the unknown unknowns" — where "unknown unknowns" means what's in the linguistic vicinity of a problem, not what's absent from the model's training. The practical implication: LLM-generated sub-question lists should always be supplemented by a deliberate adversarial pass asking "what am I not asking that I should be?" — a question the LLM cannot reliably answer from within its own knowledge structure.

## Practical implications for kit procedures

1. **Framework-selection step before decomposition:** Before generating sub-questions, classify the research type (intervention, qualitative/experience, mechanism, exposure/risk, qualitative mixed-methods). Route to the corresponding decomposition template (PICO, SPIDER, CIMO, etc.). This converts vague prompts into structured dimension-checklists with explicit blanks for missing information.

2. **Reference-interview intake pass:** Treat the initial prompt as provisional. Before any sub-question generation, run a structured probing sequence: What specific aspect of X? For what purpose / use case? What partial answers already exist? What context constrains the scope? What would make the output useful vs. not useful? Cap at 3-5 probes. Do not proceed to decomposition until probing resolves these.

3. **Enumerate dimensions, not just content:** Effective decomposition is dimensional (what domain axes must be covered) before it is content-specific (what specific questions to ask). A vague prompt about "nutrition" should first enumerate dimensions: mechanisms, populations, interventions, timescales, confounds, dissenting evidence, practical applications — before generating specific questions within each dimension.

4. **Adversarial unknown-unknowns pass:** After generating sub-questions, run a separate pass explicitly asking: "What questions am I NOT asking that would be important? What are the adjacent fields or alternative framings I haven't covered? What would a domain expert say is missing from this list?" This cannot be run by the same agent that generated the initial list — anchoring from the prior generation biases the adversarial pass.

5. **Ambiguity-first, expansion-second:** For prompts with high ambiguity (multiple plausible interpretations), the first step is to enumerate interpretations explicitly before decomposing any one of them. Skipping this step causes expansion to commit to one interpretation and narrow coverage — the SIGIR 2025 failure mode.

6. **Scope decomposition triggers intake, not output:** When a dimension is blank after the framework-selection step, this is a required intake question — not a signal to infer a default. LLMs infer missing requirements correctly only 41% of the time; treating blank dimensions as inferrable creates 59% systematic failure rate.
